## 🔴 HALLAZGOS CRÍTICOS (P0)

### 1. [P0-1] ANON_KEY Expuesta en wrangler.toml

**Severidad:** 🟡 ALTO  
**Archivo:** `wrangler.toml` línea 18  
**Problema:**
```toml
SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

**Por qué es peligroso:**
- ✅ ANON_KEY está diseñada para ser pública
- ❌ Estar en git expone históricamente
- ❌ Cualquiera que clone repo tiene acceso
- ✅ Contención: RLS debe estar habilitada

**Corrección:**
Mover a variables de entorno en `wrangler.toml`:
```toml
[vars]
SUPABASE_URL = "https://dkryjxgjqmikrhusxsau.supabase.co"
# SUPABASE_ANON_KEY → variable de entorno privada
```

Configurar en Cloudflare:
```bash
wrangler secret put SUPABASE_ANON_KEY
# Pegar valor cuando se pida
```

**Estado:** ⏳ PENDIENTE CORRECCIÓN

### 2. [P0-2] 🔴 JWT NO VALIDADO — SPOOFING DE SESIÓN

**Severidad:** 🔴 **CRÍTICO**  
**Archivo:** `_worker.js` línea 189  
**Función:** `verificarTokenSupabase()`

**Código Vulnerable:**
```javascript
async function verificarTokenSupabase(token) {
  const parts = token.split('.');
  if (parts.length !== 3) return null;

  // ❌ PROBLEMA: Decodifica sin validar firma
  const payload = JSON.parse(atob(parts[1]));
  
  // Solo valida expiración, no que fue emitido por Supabase
  const ahora = Math.floor(Date.now() / 1000);
  if (payload.exp && payload.exp < ahora) return null;
  
  return { email: payload.email, sub: payload.sub, ... };
}
```

**Por qué es crítico:**
1. Un atacante puede crear un JWT falso en su navegador:
   ```javascript
   // Atacante crea:
   const fake = btoa(JSON.stringify({ email: "admin@salon.com", exp: 9999999999 }));
   const jwt = "header." + fake + ".signature";
   ```

2. El Worker lo decodificaría y lo usaría como válido
3. Puede suplantar a cualquier usuario, incluyendo admin

**Ataque de prueba:**
```javascript
// En la consola del navegador del atacante:
const payload = btoa(JSON.stringify({ 
  email: "owner@salon.com", 
  sub: "user_123",
  exp: Math.floor(Date.now()/1000) + 86400 
}));
const fakeJWT = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9." + payload + ".fakesig";

// Enviar con Authorization header:
fetch("/api/cliente/reservas", {
  headers: { Authorization: `Bearer ${fakeJWT}` }
});
// ✅ Respuesta: Datos de owner@salon.com
```

**Corrección (JWKS Validation):**

Reemplazar `verificarTokenSupabase()` con validación de firma contra JWKS de Supabase:

```javascript
async function verificarTokenSupabase(token) {
  if (!token) return null;

  try {
    const parts = token.split('.');
    if (parts.length !== 3) return null;

    // 1️⃣ Decodificar header para obtener algoritmo y kid
    const header = JSON.parse(atob(parts[0]));
    const payload = JSON.parse(atob(parts[1]));

    // 2️⃣ Verificar expiración
    const ahora = Math.floor(Date.now() / 1000);
    if (payload.exp && payload.exp < ahora) return null;

    // 3️⃣ Obtener JWKS de Supabase (cachear por 1 hora)
    const jwksUrl = "https://dkryjxgjqmikrhusxsau.supabase.co/auth/v1/jwt";
    const jwkRes = await fetch(jwksUrl);
    if (!jwkRes.ok) throw new Error("JWKS fetch failed");
    
    const jwks = await jwkRes.json();
    const key = jwks.keys.find(k => k.kid === header.kid);
    if (!key) throw new Error("Key ID not found");

    // 4️⃣ Validar firma con crypto.subtle
    const verifyKey = await importJWK(key);
    const signature = new Uint8Array(
      token.split('.')[2].match(/.{2}/g).map(b => parseInt(b, 16))
    );
    
    const valid = await crypto.subtle.verify(
      header.alg,
      verifyKey,
      signature,
      new TextEncoder().encode(parts[0] + '.' + parts[1])
    );

    if (!valid) throw new Error("Signature invalid");

    return {
      email: payload.email || payload.user_metadata?.email,
      sub: payload.sub,
      iat: payload.iat,
      exp: payload.exp,
    };
  } catch (e) {
    console.error('JWT validation error:', e.message);
    return null;
  }
}
```

**O Alternativa Rápida (Recomendada):**
Usar Supabase SDK en el Worker para validar directamente:
```javascript
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(env.SUPABASE_URL, env.SUPABASE_SERVICE_KEY);

async function verificarTokenSupabase(token) {
  const { data, error } = await supabase.auth.getUser(token);
  if (error || !data.user) return null;
  return {
    email: data.user.email,
    sub: data.user.id,
    iat: Math.floor(data.user.created_at / 1000),
    exp: Math.floor((data.user.created_at + 3600) / 1000),
  };
}
```

**Estado:** 🔴 CRÍTICO — REQUIERE CORRECCIÓN INMEDIATA

### 4. [P1-1] Rate Limiting — PARCIALMENTE IMPLEMENTADO

**Severidad:** 🟡 ALTO  
**Archivo:** `_worker.js` línea 212  
**Estado:** `env.RATE_KV` es opcional

```javascript
async function checkRateLimit(env, key, maxReq, windowMs) {
  if (!env.RATE_KV) return true; // ⚠️ Sin KV, no hay rate limit
  ...
}
```

**Corrección:**
En `wrangler.toml`, habilitar KV:
```bash
wrangler kv:namespace create RATE_KV
```

Luego en `wrangler.toml`:
```toml
[[kv_namespaces]]
binding = "RATE_KV"
id      = "tu_kv_namespace_id"
```

### 5. [P1-2] CORS & Security Headers

**Severidad:** 🟡 MEDIO  
**Hallazgo:** Necesita verificación en `corsHeaders()`

Revisar que no hay `Access-Control-Allow-Credentials: true` innecesariamente.

```javascript
function corsHeaders(origin) {
  return {
    'Access-Control-Allow-Origin': origin || '*',
    'Access-Control-Allow-Methods': 'GET, POST, DELETE, OPTIONS',
    'Access-Control-Allow-Headers': 'Content-Type',
    // ⚠️ Ver si Credentials son necesarias
    'Access-Control-Allow-Credentials': 'true',
  };
}
```

**Recomendación:**
```javascript
function corsHeaders(origin, allowCredentials = false) {
  const headers = {
    'Access-Control-Allow-Origin': origin || '*',
    'Access-Control-Allow-Methods': 'GET, POST, DELETE, OPTIONS',
    'Access-Control-Allow-Headers': 'Content-Type, Authorization',
  };
  if (allowCredentials) {
    headers['Access-Control-Allow-Credentials'] = 'true';
  }
  return headers;
}
```
